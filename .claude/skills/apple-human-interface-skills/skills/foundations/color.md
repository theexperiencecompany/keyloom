# Color

> Using color deliberately to communicate status, express brand, convey hierarchy, and provide continuity — not as decoration.

**When to use it:** Establishing a brand accent and neutral scale, signaling status and feedback, and building a meaning-based color system that adapts to light, dark, and high-contrast.

**Guidelines**
- Give each color one meaning; don't reuse the interactive accent on non-interactive text.
- Define colors by purpose (text, secondary text, separator, link, background levels), not by their raw appearance.
- Plan light, dark, and higher-contrast variants for every color rather than relying on a single value.
- Establish a background hierarchy with primary, secondary, and tertiary surfaces to suggest depth and grouping.
- Apply the accent sparingly — reserve it for primary actions or key status, ideally one prominent action per area.
- Over busy or media-rich backgrounds, prefer simple monochromatic controls for legibility.
- Be mindful of cultural color meaning wherever color carries meaning.

**Accessibility**
- Never rely on color alone — pair it with text, icon, or shape so the information survives for colorblind users.
- Meet contrast: about 4.5:1 for normal text and 3:1 for large text and interface elements.
- Validate accent-on-background and text-on-translucency in every appearance mode.
- Respect high-contrast preferences by strengthening color and separation.

**Avoid**
- Reusing one color for conflicting meanings.
- Color as the only signal for state, errors, or interactivity.
- Over-tinting many controls at once, which kills emphasis.
- Low-contrast text on tinted or translucent surfaces.

**Full reference:** [Apple HIG](https://developer.apple.com/design/human-interface-guidelines/color)
