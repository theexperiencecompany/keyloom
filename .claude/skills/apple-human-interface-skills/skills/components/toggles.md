# Toggles

> A toggle lets people choose between a pair of opposing states, using a distinct appearance for each state.

**When to use it:** Use a switch to turn a single setting on or off immediately, a checkbox for opt-in or multi-select, and radio buttons for a few mutually exclusive options.

**Guidelines**
- Use a toggle only to represent state; for picking from a list, use a menu instead.
- Clearly identify what the toggle affects with a visible, descriptive label.
- Make the on and off states obvious through shape, fill, or a checkmark, not color alone.
- Use a mixed or indeterminate state for a checkbox that represents a group with mixed children.
- Use a switch when a change applies instantly, and a checkbox when changes apply on submit.
- Keep radio groups visually grouped with a clear heading and consistent spacing.

**Accessibility**
- Convey on and off (or checked and unchecked) states by more than color alone.
- Give every toggle a clear, descriptive accessible name.
- Provide a visible focus indicator and a logical order within groups.
- Ensure the hit target meets roughly 44pt, and extend the tappable area to the label.
- Maintain strong contrast for the control in each of its states.

**Avoid**
- Using a switch to trigger an action rather than represent state.
- Conveying state with color only.
- A long list of radio buttons where a menu fits better.
