# Search Fields

> A search field is an editable text field, with a search icon and a clear button, that lets people search content for terms they enter.

**When to use it:** Use a search field to query or filter a collection by typed terms — inline when scoped to a single view, or in a header when search is global.

**Guidelines**
- Show a search icon and a clear button so the field is instantly recognizable and easy to reset.
- Use placeholder text to communicate scope, not as the field's only label.
- Search as the person types when feasible, so results refine continuously.
- Show recent searches before typing and predictive suggestions while typing.
- Order results by relevance, most-likely first, and categorize them when that aids scanning.
- Offer scope filters, defaulting to a broader scope and letting people narrow it.
- On a dedicated search view, consider focusing the field on arrival — but avoid this where an on-screen keyboard would cover the content.

**Accessibility**
- Give the field a clear, descriptive label so its purpose is obvious without relying on the icon alone.
- Convey result counts and changes so they're perceivable without sight.
- Keep suggestions navigable in a logical focus order.
- Ensure the clear button and the field meet an adequate touch target of roughly 44pt.
- Maintain sufficient contrast for placeholder text, icons, and entered text.

**Avoid**
- A bare input with no visual search affordance.
- Suggestion lists that can't be reached or operated in a logical order.
- Forcing the keyboard open over results on small screens.
