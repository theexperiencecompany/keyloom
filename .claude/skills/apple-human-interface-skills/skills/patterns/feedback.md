# Feedback

> Cues that tell people what's happening, the result of an action, and how to avoid or fix mistakes.

**When to use it:** For communicating ongoing status, confirming success or failure of an important action, warning before irreversible actions, or offering a way to correct a problem.

**Guidelines**
- Match the delivery to the significance: passive, inline cues for routine status; an interrupting prompt only for critical warnings.
- Convey feedback through more than one channel (text plus icon and color, not color alone).
- Place status near the items it describes so people see it in context.
- Reserve interrupting alerts for critical, ideally actionable information; overuse dilutes their meaning.
- Warn before unexpected, irreversible data loss, but not for expected outcomes (for example, emptying the trash).
- Confirm completion only for significant actions; when a command can't run, say so and offer a path to resolve it.

**Accessibility**
- Announce dynamic feedback to people using assistive technology, gently for status and more assertively for urgent messages.
- Never rely on color alone; pair it with legible text and an icon.
- Present error messages next to the field or item they describe.
- Don't dismiss important feedback so quickly that people can't read it.

**Avoid**
- Interrupting with a prompt for low-importance status.
- Color-only success or error states.
- Silent failures, or generic errors with no cause or next step.
