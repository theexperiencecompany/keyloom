# Offering Help

> Contextual help — tips, hints, and links — offered when an interface isn't fully self-explanatory.

**When to use it:** For a new or non-obvious feature, a control whose purpose needs a brief explanation, or a complex task that warrants a short tutorial or a link to more information.

**Guidelines**
- Match help to the task: a short inline note for simple actions, a brief tutorial for multistep goals.
- Relate help to what the person is doing right now, and make it easy to dismiss or ignore.
- Use tips for simple features (a rule of thumb is three actions or fewer); keep them short and action-oriented.
- Use wording suited to the platform and input method, and avoid promotional content.
- Show tips only to people who'd benefit (not those already using the feature), and cap how often they appear.
- Don't explain standard controls; describe what they do specifically in your app, and link to relevant settings or resources.

**Accessibility**
- Make help reachable without a pointer (on focus, not hover only) and clearly associated with its control.
- Ensure tips and coachmarks can be dismissed without a pointer and are announced to people using assistive technology.
- Don't convey help solely through an icon; pair it with legible text, and keep contrast adequate.

**Avoid**
- Bloated help that re-teaches standard interface or input behavior.
- Hover-only hints that keyboard and touch users can't trigger.
- Promotional or off-topic content inside a tip.
